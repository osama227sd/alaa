Introduction

Software Design

The goal of the design phase is to transform the requirements specified in the SRS document into a structure that is suitable for implementation in some programming language.

Design focuses on the module view.

Levels

Design Levels

Software design process often has two levels:

  • High-level design
  • Detailed design

High-level design identifies modules, control relationships, and interfaces; output is Software Architecture.

Detailed design focuses on data structures and algorithms; output is Module Specification Document.

Methods

Software Design Methods

  • Structured Design Approach
  • Object-Oriented Design Approach
Principles

Design Objectives & Principles

The goal is to find the best possible design within requirements and constraints.

Problem Partitioning

Divide system into Input, Processing, Output.

Abstraction

Describe component behavior without internal details.

Top-Down & Bottom-Up

Top-down: system ‚Üí components; Bottom-up: components ‚Üí system.

Architecture

Architectural Design

Large systems decomposed into subsystems. Key metrics:

  • Depth ‚Äì levels of components
  • Width ‚Äì components per level
  • Fan-in ‚Äì incoming edges
  • Fan-out ‚Äì outgoing edges
Modularity

Modular Systems

Discrete components allow independent implementation and easier maintenance, testing, and reuse.

Coupling and Cohesion

Coupling ‚Äì interdependence between modules
