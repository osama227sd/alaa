CH3 – Part 1

Shared Memory Architecture

ما هي Shared Memory Architecture؟

هي معمارية حاسوب تحتوي على أكثر من Processor، وجميع الـ Processors يشتركون في نفس مساحة الذاكرة (Shared Memory). أي Processor يستطيع الوصول إلى أي بيانات موجودة في الذاكرة.

تصنيف أنظمة Shared Memory

تنقسم أنظمة Shared Memory إلى ثلاثة أنواع رئيسية:

  • Uniform Memory Access (UMA)
  • Non-Uniform Memory Access (NUMA)
  • Cache Only Memory Architecture (COMA)

1️⃣ Uniform Memory Access (UMA)

في هذا النوع يكون زمن الوصول إلى أي مكان في الذاكرة متساويًا لجميع الـ Processors.

لا يوجد Processor مميز عن الآخر، لذلك تسمى هذه الأنظمة: Symmetric Multiprocessors (SMP).

➤ كل المعالجات تصل للذاكرة بنفس السرعة.

2️⃣ Non-Uniform Memory Access (NUMA)

كل Processor يكون متصل بجزء من الذاكرة الخاصة به (Local Memory). زمن الوصول للذاكرة يعتمد على المسافة بين الـ Processor والذاكرة المطلوبة.

➤ Local Memory أسرع من Remote Memory.

3️⃣ Cache Only Memory Architecture (COMA)

في هذا النظام لا توجد ذاكرة رئيسية تقليدية، بل يتكون النظام بالكامل من Caches.

البيانات تتحرك (migrate) إلى الـ Processor الذي يطلبها.

Bus-Based Symmetric Multiprocessors

يعتبر أبسط شكل من أشكال ربط المعالجات، حيث تتصل جميع الـ Processors بالذاكرة عن طريق Bus واحد مشترك.

المشكلة الأساسية في هذا التصميم هي تشبع الـ Bus عند محاولة أكثر من Processor الوصول للذاكرة في نفس الوقت.

➤ استخدام Cache يقلل الضغط على الـ Bus.

Basic Cache Coherency

تكون الـ Caches متوافقة (Coherent) عندما تكون جميع النسخ من نفس البيانات في كل الكاشات لها نفس القيمة.

Write-Through Policy

عند أي تعديل في الكاش، يتم تحديث الذاكرة الرئيسية فورًا.

➤ بسيط لكنه يسبب ضغط كبير على الـ Bus.

Write-Back Policy (شرح مهم)

في هذه السياسة يتم التعديل داخل الكاش فقط، ولا يتم تحديث الذاكرة الرئيسية مباشرة.

يتم تحديث الذاكرة فقط عند خروج (استبدال) البلوك من الكاش.

➤ يعتمد على Dirty Bit لمعرفة البلوك المعدّل.
